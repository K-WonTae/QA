# 결재·회계 테이블 (Resolution* / Acct*)

> 출처: `D:\DeskList\업무\행정` 내 지출결의서 목록·개인별 지급내역·계정코드 쿼리 분석
> 공통코드 매핑은 [knowledge/행정/결재](../결재/index.md) 참조.

지출결의(전자결재)와 회계 계정을 관리하는 도메인. 결의 키는 `AcctDivCode`(계정구분) + `Year`(회계연도) + `ResolutionNO`(결의번호). 소속명 조회: `sf_cor_sosognm_johoe`.

---

## ResolutionMaster — 지출결의서 마스터
지출결의서 1건의 헤더(결의구분·금액·결재상태).

| 컬럼 | 설명 | 공통코드 |
|---|---|---|
| ResolutionNO | 결의번호 (PK 일부) | |
| AcctDivCode | 계정구분코드 (→ AcctDivCode) | |
| Year | 회계연도 | |
| ResolutionDiv | 결의서구분 | [결의서구분 A113](../결재/결의서구분(A113).md) |
| ResolutionDate | 결의일자 | |
| TotalAmt | 결의 총액 | |
| Subject | 적요(제목) | |
| CtrlDiv | 통제구분 | [통제여부 A110](../결재/통제여부(A110).md) |
| ProcessState | 결재진행상태 | [결재진행상태 A109](../결재/결재진행상태(A109).md) |
| ConfirmYN | 확정여부 | |
| SlipNO | 전표번호 | |
| ReturnDesc | 반려 사유 | |
| OrgID | 기안 소속 | (Organization) |

## ResolutionDetail — 지출결의서 상세
결의서 명세 행(계정·금액·지급처). `ResolutionMaster`와 `AcctDivCode + Year + ResolutionNO`로 연결.

| 컬럼 | 설명 |
|---|---|
| ResolutionNO / AcctDivCode / Year | 결의 키 |
| AcctCode | 계정코드 (→ AcctCode) |
| Amt | 금액 |
| (지급처) | PaymentCompany/Customer |

## ResolutionMngrCode — 결의 관리코드(담당)
결의별 관리부서·담당자(관리코드) 매핑.

| 컬럼 | 설명 |
|---|---|
| AcctDivCode / Year / ResolutionNO | 결의 키 |
| MngrCode / MngrName | 관리코드 / 관리(담당)명 |

## AcctCode — 계정코드
회계 계정과목 마스터(계정코드·계정명·계층).

## AcctDivCode — 계정구분코드
회계 단위(법인/교비/산학 등) 구분. 컬럼: AcctDivCode, AcctDivName.

## BussCode — 업무(사업) 코드
회계연도·계정구분별 사업(업무) 코드. 컬럼: AcctDivCode, Year, BussCode, BussName. (자산 취득과 공유 — [자산·구매](자산·구매.md))

## PaymentCompany / Customer — 지급처 / 거래처
지급 대상(업체)·거래처 정보(계좌·사업자번호).

---

### 연관
- 전자결재 연동 배치는 `99.전자결재연동`(KON_Adbatch) 참조. `groupuser`(그룹사용자: userid, groupid)는 결재선/권한 그룹.
