# knowledge 폴더 문서 추가 가이드

이 프로젝트의 `knowledge` 폴더는 QA가 답변할 때 참고하는 지식 문서 저장소입니다. 문서를 추가할 때는 "어느 업무 영역의 어떤 주제인지"가 폴더와 파일명만 봐도 드러나게 정리하는 것이 가장 중요합니다.

## 기본 구조

현재 기본 구조는 다음과 같습니다.

```text
knowledge/
  공통/
  학사/
    등록/
    수업/
    입학/
    장학/
    테이블/
    학사/
    학적/
  행정/
    결재/
    구매/
    급여/
    기부/
    인사/
    자산/
    테이블/
    통계/
```

새 문서는 가능하면 기존 분류 아래에 넣습니다. 예를 들어 등록 관련 코드는 `knowledge/학사/등록/`, 인사 관련 코드는 `knowledge/행정/인사/`에 둡니다.

## 파일 형식

기본 지식 출처는 `.md` 파일을 읽도록 설정되어 있습니다. 새 지식은 특별한 이유가 없으면 Markdown 파일로 작성합니다.

권장 파일명:

```text
업무명(코드).md
```

예시:

```text
knowledge/학사/등록/등록구분(SI01).md
knowledge/행정/급여/급여구분(A519).md
knowledge/공통/은행코드(C101).md
```

공통 설명이나 목록 파일은 `index.md`를 사용합니다.

```text
knowledge/학사/등록/index.md
knowledge/행정/인사/index.md
```

## 문서 작성 형식

각 문서는 아래 형식을 권장합니다.

```md
# 문서 제목(코드)

## 업무 설명
이 코드나 문서가 어떤 업무에서 사용되는지 적습니다.

## 주요 코드
- CODE1 코드명: 설명
- CODE2 코드명: 설명

## 관련 업무
- 메뉴명 또는 업무 흐름
- 연결되는 테이블 또는 화면

## 검색어
- 사용자가 질문에 쓸 만한 표현
- 약어
- 동의어
```

검색어를 넣어두면 질문과 문서가 더 잘 연결됩니다. 사용자가 실제로 말할 표현을 함께 적는 것이 좋습니다.

## 새 문서 추가 절차

1. 적절한 폴더를 고릅니다.

   예: 장학 관련 문서라면 `knowledge/학사/장학/`

2. 새 `.md` 파일을 만듭니다.

   예: `knowledge/학사/장학/장학상태(SH99).md`

3. 제목, 업무 설명, 주요 코드, 검색어를 작성합니다.

4. 같은 폴더의 `index.md`가 있다면 새 문서를 목록에 추가합니다.

   예:

   ```md
   - SH99 장학상태
   ```

5. 앱을 다시 실행하거나 새 질문을 보내 문서가 검색되는지 확인합니다.

## 새 폴더를 만들 때

기존 분류에 맞지 않는 업무라면 새 하위 폴더를 만들 수 있습니다.

예시:

```text
knowledge/학사/졸업/
  index.md
  졸업구분(SG01).md
  졸업사정상태(SG02).md
```

새 폴더를 만들면 `index.md`를 함께 만들어 그 폴더의 문서 목록과 대표 검색어를 정리합니다.

## 좋은 문서의 기준

- 제목에 업무명과 코드가 함께 들어간다.
- 본문 첫 부분에 이 문서가 어떤 업무를 설명하는지 적는다.
- 코드값만 나열하지 않고 사용 맥락을 함께 적는다.
- 사용자가 질문할 법한 표현을 `검색어`에 넣는다.
- 한 파일에는 한 주제만 담는다.

## 피해야 할 것

- 비밀번호, 토큰, 개인정보, 실제 주민번호 같은 민감 정보를 넣지 않습니다.
- 너무 큰 원본 문서를 그대로 붙여넣지 않습니다.
- 같은 내용을 여러 파일에 중복해서 넣지 않습니다.
- 파일명을 `새문서.md`, `정리.md`처럼 모호하게 만들지 않습니다.
- DB 파일, 로그 파일, 캐시 파일은 `knowledge`에 넣지 않습니다.

## Git에 반영하기

문서를 추가한 뒤에는 아래 순서로 커밋합니다.

```powershell
git status --short
git add knowledge
git commit -m "Update knowledge documents"
git push
```

특정 문서만 올리고 싶다면 `git add knowledge` 대신 파일 경로를 직접 지정합니다.

```powershell
git add "knowledge/학사/장학/장학상태(SH99).md"
git commit -m "Add scholarship status knowledge"
git push
```

## 추가 예시

새로 `휴복학 처리 상태` 문서를 추가한다면:

파일 경로:

```text
knowledge/학사/학적/휴복학처리상태(SB99).md
```

내용:

```md
# 휴복학처리상태(SB99)

## 업무 설명
학생의 휴학, 복학 신청이 어떤 처리 단계에 있는지 관리하는 코드입니다.

## 주요 코드
- 10 신청: 학생 또는 담당자가 신청을 등록한 상태
- 20 접수: 담당 부서가 신청을 접수한 상태
- 30 승인: 최종 승인된 상태
- 90 반려: 요건 미충족 등으로 반려된 상태

## 관련 업무
- 학적 변동
- 휴학 신청
- 복학 신청

## 검색어
- 휴학 상태
- 복학 상태
- 휴복학 처리
- 학적변동 진행상태
```
